🎯 1. Overview
DZ-Ghost Pro is a WordPress/WooCommerce plugin specifically designed for the Algerian market (Cash on Delivery - COD). It transforms a standard WooCommerce store into an integrated operations center that combines:
One-page COD checkout form
Order confirmation center for call center workers (Confirmation Desk / Worker Desk)
Smart shipping tracking system with 22+ Algerian shipping companies
Server-side advertising tracking (CAPI) for Facebook, TikTok, and Snapchat
Advanced analytics dashboard (Dashboard Pro)
Financial and accounting system
Anti-bot and anti-fraud protection
--------------------------------------------------------------------------------
🏗️ 2. Architecture
2.1 Architecture Philosophy
The plugin is built on a Drop-In Feature Architecture where each feature is an independent module (Feature Module) that can be added or removed without affecting the rest of the system.
graph TD
    A["dz-monster-checkout.php<br/>Main Entry Point"] --> B["class-dz-monster.php<br/>Core Engine"]
    B --> C["class-dz-feature-loader.php<br/>Dynamic Feature Loader"]
    B --> D["class-dz-order-manager.php<br/>Order Engine"]
    B --> E["class-dz-bordereau-integration.php<br/>Shipping Bridge"]
    B --> F["class-dz-security.php<br/>Security System"]
    B --> G["class-dz-license-manager.php<br/>License Management"]
    
    C --> H["admin/features/*<br/>25+ Admin Modules"]
    C --> I["public/features/*<br/>Frontend Modules"]
    
    E --> J["woo-bordereau-generator/<br/>Shipping Bordereau Engine"]
2.2 Design Patterns Used
Pattern
Usage
Singleton
DZ_Bordereau_Integration, DZ_Slicer, DZ_Cache
Feature Interface
Each feature implements DZ_Feature interface
Hook-Only Communication
Component communication via WordPress Hooks only
Drop-In Loading
Automatic feature discovery via init.php in each folder
Lazy Evaluation
Availability check performed only once per request
2.3 Restricted Mode
When the license is not activated, the plugin operates in restricted mode where:
Only the license activation page is displayed
Any attempt to access other pages is automatically redirected
No public or admin features are loaded

🛡️ 8. Security Layers
8.1 Order Protection (Checkout Security)
Algerian Phone Validation — Only accepts 05/06/07 with rejection of repeated numbers
IP Order Limit — Maximum 3 orders per day per IP
VPN/Proxy Detection — HTTP header analysis
Blacklist — Block by IP, phone, or email
Spam Detection — Filter links and suspicious keywords in fields
Quantity Limit — Reject orders with more than 100 units
8.2 Tracking Protection (Tracking Security)
Anti-Bot Shield — 20+ known bot signatures
AJAX Filtering — Skip bot checks in CAPI context
Admin Exclusion — Don't track admins to protect data accuracy
8.3 Infrastructure Protection
AES-256-CBC — Encrypt all sensitive data
CSP Headers — Prevent Clickjacking and XSS
Rate Limiting — Protect against Brute Force attacks
Nonce Verification — Protect AJAX from CSRF
HPOS Compatibility — Compatible with WooCommerce's new order system

⚠️ 12. Technical Notes
[!NOTE]
Development Files — There are development/test files in the main folder (test.php, test_anderson.php, dz_debug_orders.log, tmpclaude-*) that should be deleted before production
Global Error Handler — There's a custom set_error_handler in the entry point to catch Deprecation notices — may need review in production
Code Size — 122K+ lines of PHP indicates a large project — periodic code review for dead code is recommended
dz.json File — 6.7 MB large geographic file — consider loading on demand instead of always including
